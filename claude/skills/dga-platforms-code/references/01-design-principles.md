# 01 — Design principles

These principles guide every decision. When trade-offs arise, resolve them in favor of trust,
clarity, and accessibility.

## Government-grade trust

The interface represents a public authority. It must look official, stable, and credible: restrained
color, clear structure, predictable behavior, accurate content. Avoid anything that reads as
promotional, playful, or experimental.

- Prefer convention over cleverness.
- Show the entity's role and the service's official nature through tone, not decoration.
- Never fabricate official outcomes, seals, or statuses.

## Service-first UX

Every screen exists to move the user through a government service. Make the task and the next step
obvious.

- Lead with the task, not the brand.
- State what the service does, what is required, and what the outcome will be.
- One primary action per view; everything else is secondary.

## Accessibility-first

Accessibility is a requirement, designed in from the start — not a later cleanup.

- Sufficient contrast, visible focus, full keyboard support, semantic structure.
- Status and meaning never depend on color alone.
- Works with assistive technology and on real devices.

## Arabic RTL correctness

The default audience reads Arabic, right to left. RTL is baseline correctness, not an enhancement.

- Correct direction, alignment, and mirroring.
- Correct handling of mixed Arabic/Latin content, numbers, and codes.
- Natural, idiomatic Arabic — not literal translation.

## Consistency over novelty

Consistency builds trust and lowers cognitive load. Reuse beats reinvention.

- Reuse established patterns, components, and tokens.
- Keep terminology, layout, and behavior consistent across the product.
- Do not introduce one-off visual treatments.

## Minimal, calm, official visual language

Calm interfaces feel trustworthy. Reduce visual noise.

- Generous whitespace, clear hierarchy, limited type and color variety.
- Restrained, purposeful motion.
- Let structure and typography carry the design.

## No unnecessary decoration

Every element earns its place by serving the task.

- No decorative gradients, illustrations, or animation that don't aid comprehension.
- No purely aesthetic embellishment that competes with content or the primary action.
- If an element doesn't help the user complete the service, remove it.

## Future outlook (vision note)

The Platforms Code Guide v1.0 mentions future support for innovative digital channels and
technologies such as virtual reality, augmented reality, artificial intelligence, generative AI, and
automation of design and development processes.

This is a **vision note only**. Do not turn it into implementation requirements unless the user
explicitly asks.
