Foundations

Theming

Platforms Code is a unified national design system — the default accent is the Saudi green. When a service or section genuinely needs to be distinguished, you can switch only the primary accent to another verified palette via an opt-in data-pc-theme preset. Everything else stays the same.

⚖️

Accent only — no invented palettes. Themes reassign the primary accent to a verified palette (SA / Gold / Lavender). Semantic colors (success/error/warning/info), neutrals, typography, spacing, and the green chart scheme stay fixed. Prefer the default green unless there's a clear reason to differentiate.

Usage

Set data-pc-theme on a themed root (typically the same element with data-pc-root). No attribute = the default green.

<main data-pc-root dir="rtl" lang="ar" data-pc-theme="gold">
  <Button variant="primary">ابدأ الخدمة</Button>   {/* gold accent, dark text */}
</main>

Guardrails

Do

  • Switch the accent only to a verified palette (SA / Gold / Lavender).
  • Keep semantics, neutrals, type, spacing, and charts fixed across themes.
  • Preserve contrast — use dark text on gold; keep focus rings ≥ 3:1.
  • Keep one primary action per view.
  • Prefer the default green unless there's a clear reason to differentiate.

Don't

  • Invent a per-service brand hue outside the verified palettes.
  • Let the accent recolor success/error/warning/info or the chart greens.
  • Use white text on a gold accent (fails contrast at ~2.3:1).
  • Theme purely for decoration or to manufacture emphasis.
